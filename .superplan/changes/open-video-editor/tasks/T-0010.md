## Description
Use the fixed fixtures to prove the thin AVFoundation/VideoToolbox/CoreVideo bridge can retain decoded YUV surfaces through a Metal color/effect pass, owned RGBA texture, bridge-owned frame lease, and Qt render-thread presentation with explicit color, synchronization, cancellation, and lifetime semantics.

## Acceptance Criteria
- [ ] Capability probing reports the actual VideoToolbox hardware/software path, accepted codec profile, native pixel format, color metadata, and failure reason.
- [ ] CVPixelBuffer NV12/P010 output reaches CVMetalTexture plane views, a validated YUV/range/transfer-to-linear-working-RGBA Metal pass, an owned 2D RGBA texture, and Qt render-thread import with zero steady-state GPU-to-CPU frame transfers after warm-up.
- [ ] The bridge-owned frame lease retains the pixel buffer, plane views, RGBA texture, device generation, and completion primitive through actual GPU completion; rapid seeks, multi-monitor/resize/DPI changes, project close, and repeated teardown produce no stale presentation or leaks.
- [ ] QHD-I original and 3K-I proxy playback meet the vertical-slice gates; CPU-oracle samples verify Rec.709 primaries, matrix, limited/full range, transfer, black/white, and source order while traces count transfers, allocations, dropped frames, owned budgets, RSS, and unified-memory use.
- [ ] Unsupported or corrupt media and unavailable hardware decode return typed diagnostics and a disclosed fallback without crash or silent backend substitution.

## Verification

- Run the full macOS development fixture corpus under Metal validation, sanitizers where applicable, and 100 seek-cancellation cycles.
- Inspect GPU/copy traces and retain a failing trace if any readback or lifetime violation occurs.
- Record the exact Mac model, OS, VideoToolbox path, Metal feature set, power/thermal state, and raw samples.
