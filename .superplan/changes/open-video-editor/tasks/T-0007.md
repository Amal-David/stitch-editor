## Description
Implement the UI- and codec-independent Rust source of truth: normalized exact time, immutable revisioned project model, typed commands, deterministic undo/redo, canonical serialization and hashing, asset identity, and transactional SQLite persistence/recovery. Do not add media decoding or Qt concerns to this task.

## Acceptance Criteria
- [ ] Normalized rational time, half-open ranges, named rounding policies, VFR PTS retention, checked overflow, and the absolute-time platform-adapter contract have boundary/property tests and 10,000-frame 23.976/29.97/59.94 vectors.
- [ ] The schema-versioned editorial model and typed command reducer produce canonical bytes and stable revision hashes independent of insertion order or process restart.
- [ ] One thousand seeded edit/replay/undo/redo sequences end at their expected hashes and reject invalid preconditions without partial mutation.
- [ ] `ProjectStore v1` bundles SQLite >= 3.51.3, verifies compile options/journal mode/`synchronous=FULL`, uses one writer, acknowledges only an atomically committed command+revision+head, and treats database/WAL/SHM as one live unit.
- [ ] Fault injection before commit, after WAL commit but before response, during snapshot, checkpoint, compaction, and archive preserves the durable protocol: pre-commit failure exposes no receipt and leaves the prior head; post-commit ambiguity retains the new head and receipt so retrying the same `RequestId` returns it without reapplying; no returned receipt is lost and no timeline is corrupt or silently repaired.
- [ ] The crate boundary contains no Qt, WebView, decoder, encoder, GPU, or OS media ownership.

## Verification

- Run unit, property, serialization-golden, migration, 10,000-frame time-vector, and the full ProjectStore crash matrix under debug and release builds.
- Run platform-independent vectors on both macOS and Windows and compare canonical hashes.
- Record test commands, seeds, fixture digests, failures, and timings as review evidence.

Current evidence and its explicit Windows/power-loss limits are recorded in
[`../research/review-core-store.md`](../research/review-core-store.md). The task
remains open until the Windows hash comparison runs.
