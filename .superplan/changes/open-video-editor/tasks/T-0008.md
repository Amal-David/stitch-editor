## Description
Build the licensed deterministic 30-second development and 10-minute release fixtures, canonical project recipe, independent correctness oracles, structured traces, render manifests, and reproducible cold/warm benchmark runner defined by the vertical-slice contract.

## Acceptance Criteria
- [ ] The QHD-I, QHD-LGOP, 3K-I, optional 3K-LGOP variants, and VFR-AV generator recipes produce the exact 8-bit AVC/AAC baseline plus companion WAV oracles and versioned manifests pinning profile/level, bitrate, GOP, delay/padding, timescale, sample entry, hashes, timestamps, markers, and license data.
- [ ] A checked-in `oracle-lock` pins independently implemented generator, CPU reference, demux/decode/container verifier, metadata checker, tolerance set, and versions/hashes; it detects wrong frames, stale generations, timing drift, sample-count errors, non-monotonic PTS, and incorrect color metadata.
- [ ] The harness emits raw repetitions, failures, Perfetto-compatible traces, semantic/execution/mix digests, render manifests, adapter rounding errors, hardware/software codec paths, owned resource-token/queue metrics, process memory observations, and all required platform metadata.
- [ ] Cold and warm runs are reproducible, randomly interleaved, retain failed samples, and flag same-machine correctness or greater-than-5-percent median regressions.
- [ ] Fixture and oracle tests run without depending on the editor implementation they evaluate.

## Verification

- Regenerate the development set twice and compare canonical decoded and metadata oracles.
- Inject known one-frame, one-sample, PTS/time-map, color, audio-fade, container, and stale-generation faults and prove each independent oracle fails.
- Validate one complete disclosure bundle against the vertical-slice schema.

The mandatory AAC bitrate decision is resolved in
[`../research/codec-profile-conflict.md`](../research/codec-profile-conflict.md):
AAC-LC 48 kHz stereo at 192 kb/s is the cross-platform baseline, and 320 kb/s
is an optional capability probe.

## Current implementation status

The first implementation foundation is checked in under `tools/fixtures/`,
`tools/oracles/`, and `tools/benchmark-harness/`. It provides deterministic
streaming source recipes, duration-spanning markers, typed artifact/capability
schemas, a source-verifiable lock, isolated evidence comparisons, complete
disclosure types, seeded cold/warm scheduling, failure retention, trace JSON,
and paired regression calculations.

T-0008 remains incomplete. The independent implementation review in
[`../research/review-t0008-implementation.md`](../research/review-t0008-implementation.md)
must be cleared before review: native AVC/AAC fixture generation and capability
capture, a real independent BMFF/decode/metadata oracle with pixel/audio
tolerances, real benchmark-run collection, and end-to-end disclosure evidence
are not yet implemented.
