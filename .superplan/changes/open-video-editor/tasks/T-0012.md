## Description
Implement and falsify the Rust 48 kHz float audio graph for clip gain, gain automation, equal-power fades, channel mapping, latency compensation, preview device output, and exact offline blocks. Choose one explicit device boundary—a Rust-owned CPAL callback or minimal C trampoline into a preallocated Rust context. The callback is the preview master clock and must remain hard-real-time safe.

## Acceptance Criteria
- [ ] Fixed bounded audio blocks produce exact offline sample positions, declared channel mapping, gain ramps, equal-power crossfades, latency compensation, and sanitized finite output.
- [ ] Instrumentation proves the device callback performs zero allocations, ownership-changing commands, locks, disk/network I/O, waits, logging, or unwinding and consumes only fixed preallocated buffers plus lock-free bounded rings.
- [ ] Ten-minute QHD-I and 3K-source proxy runs plus a 16-active-track stress mix at recorded 128- and 256-frame buffers have zero underruns and calibrated A/V marker drift no greater than 5 ms on both operating systems.
- [ ] Buffer underrun, hot unplug/device reopen, device-clock discontinuity, sample-rate mismatch, cancellation, teardown, and overload produce explicit diagnostics and deterministic recovery or stop behavior.
- [ ] The same canonical audio graph and automation sampling feed preview and export; the offline VFR-AV oracle is exact to the sample.

## Verification

- Run impulse, chirp, fade-curve, gain-keyframe, channel-map, latency, NaN/Inf, long-run drift, and device-change tests.
- Use allocator, mutex, file/network-I/O, logger, wait, and unwind intercept counters around the callback and fail the test on the first forbidden operation.
- Retain callback timing, underrun, queue, device-offset, and A/V-marker evidence for macOS and Windows.
