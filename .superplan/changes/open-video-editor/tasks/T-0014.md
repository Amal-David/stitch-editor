## Description
Produce clean-machine macOS and Windows packages, run the complete functional, failure-injection, correctness, and performance contract, publish reproducible evidence, and decide whether ADR-001 is accepted, revised, or rejected before adding product breadth.

## Acceptance Criteria
- [ ] Signed/notarized macOS and signed Windows packages install on clean machines, open the same canonical project, preserve its revision hash, preview it, and produce oracle-valid exports.
- [ ] Every functional acceptance item and architecture gate in the vertical-slice contract has linked passing evidence or an explicit failure; no skipped assertion is reported as passed.
- [ ] QHD original and 3K-source proxy playback, refresh-relative UI p99, 16-track 128/256-buffer audio stress, A/V sync, rational-adapter error, cancellation, owned resource budgets, backpressure, zero-copy, recovery, and mandatory QHD/3K export gates pass on at least one disclosed machine per OS.
- [ ] Cold/warm, seek, long-GOP, 3K original, proxy-generation, export, resource, and package-size baselines include raw samples, failures, traces, manifests, fixture/oracle digests, and complete hardware/software metadata.
- [ ] A final architecture review accepts, revises, or rejects ADR-001 from evidence; no “most efficient” comparative claim is made without a correct identical public suite.
- [ ] The source distribution has an OSI-approved license, third-party notices, locked dependencies, SBOM, build instructions, support matrix, reproducible fixture recipes, and a dated distribution review for GPL/Qt and H.264/AAC jurisdiction/system-framework obligations without bundled FFmpeg, NodeAV, or codec binaries.
