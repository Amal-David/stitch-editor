## Description
Implement the smallest versioned Rust-to-C ABI and Qt Quick shell needed to prove ownership, command flow, immutable snapshot/diff delivery, cancellation, errors, DPI/accessibility, and a native preview item. Prove one externally created same-device 2D RGBA texture and bridge-owned frame lease; platform decoding belongs to later spikes.

## Research

- [Qt 6.11.1 native preview boundary](../research/qt-native-boundary.md)
- [C ABI implementation and verification evidence](../research/review-c-abi.md)

## Acceptance Criteria
- [ ] The C ABI uses opaque core handles and bridge-owned `frame_lease` IDs, explicit ownership/lifetime functions, POD metadata, version negotiation, device-generation/revision/epoch tags, structured errors, and no exposed Rust, C++, COM, Objective-C, Metal, D3D11, or Qt object layout.
- [ ] ABI conformance, fuzz, sanitizer, reentrancy, callback-thread, and create/destroy stress tests pass without leaks, use-after-free, unwinding, or undefined ownership.
- [ ] Qt Quick can open a project snapshot, submit a command batch, consume a revision-tagged diff, cancel obsolete work, and import one externally created same-device 2D RGBA texture only on the scene-graph render thread with explicit completion and owner-thread retirement.
- [ ] Raw video/audio buffers never cross Qt property serialization, JSON, IPC, or a WebView; the boundary can pass only native-surface handles plus synchronization metadata.
- [ ] The pinned Qt/native-interface build configures the graphics device before scene-graph initialization, asserts Metal on macOS and D3D11 on Windows, rejects software/WARP/wrong-device paths for this gate, and launches a keyboard-accessible DPI-correct shell on both OSes.

## Verification

- Run Rust, C ABI, C++/Qt, sanitizer, and repeated lifecycle tests on macOS and Windows.
- Capture boundary traces showing thread, device generation, revision, cancellation epoch, frame-lease acquire/submit/completion/retire, allocation, and ownership events through resize and teardown.
- Record package/toolchain versions and verify the selected GPL distribution path before expansion.
